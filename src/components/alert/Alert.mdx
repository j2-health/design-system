# Alert

Alerts display important messages that require user attention. They provide
contextual feedback about system status, user actions, or information that
shouldn't be missed.

## When to Use

Use alerts to communicate information that is relevant to the user's current
context or workflow. Choose the appropriate variant based on the message's
intent:

## When Not to Use

- **For inline form validation:** Use field-level error messages instead
- **For transient notifications:** Use Toast or Notification components for
  messages that auto-dismiss
- **For confirmation dialogs:** Use Modal when user action is required before
  proceeding
- **For empty states:** Use dedicated empty state patterns instead

## Best Practices

- Keep messages concise and actionable
- Use the `description` prop for additional context when needed
- Consider using `banner` mode for page-level alerts
- Make alerts dismissible (`closable`) when the information isn't critical
